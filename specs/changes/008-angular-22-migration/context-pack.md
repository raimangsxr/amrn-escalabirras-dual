# Context Pack: 008-angular-22-migration

## Goal

Bump the Angular frontend from v20 to v22 (specifically 22.0.4)
while preserving the existing NgModule + RxJS + zone.js +
Karma/Jasmine architecture. This is a tooling / dependency /
builder migration, not an architectural rewrite.

## Relevant Contracts

- `specs/contracts/frontend-angular/contract.md` (updated —
  Angular 20 → Angular 22).

## Current Understanding

- The frontend is on Angular 20.3.25 as of the close of
  `007-angular-20-migration`. `package.json` declares
  `@angular/core ~20.3.25`, `@angular/material ~20.2.14`,
  `@angular/cdk ~20.2.14`, `@angular/cli ~20.3.30`,
  `@angular/build ~20.3.30`, `typescript ~5.8.3`, `zone.js
  ~0.15.1`, `rxjs ~7.8.0`.
- `angular.json` declares four `@angular/build:*` builders
  (application, dev-server, extract-i18n, karma).
- `tsconfig.json` declares `moduleResolution: "bundler"`,
  `target: "ES2022"`, `module: "ES2022"`,
  `experimentalDecorators: true`, `strict: true`.
- `src/app/app.module.ts` bootstraps via
  `provideHttpClient(withInterceptorsFromDi())` (added in v18).
- All components carry `standalone: false` (added in v19).
- The dev environment runs Node v22.14.0, which does not meet
  Angular 22's minimum (`^22.22.3 || ^24.15.0 || >=26.0.0`).
  Homebrew has `node@24` (24.18.0) installed but not symlinked;
  the `ng update` invocations will use
  `/opt/homebrew/opt/node@24/bin/node` via `PATH`.
- Material is imported only as a prebuilt CSS file. No Material
  components, modules, or services are used in `src/app/`.
- The frontend has zero unit tests. The `test` target exists for
  parity but is not run in CI.
- Angular 21 (released November 2025) and Angular 22 (released
  May 2026) are both within or near the model's knowledge cutoff
  (January 2026). Detailed release notes for v22 are not
  available; the migration leans on `ng update` schematics to
  apply any breaking changes automatically.

## Constraints

- **NgModules stay.** No standalone components, no
  `bootstrapApplication`. `AppModule` remains the bootstrap
  module.
- **RxJS + `BehaviorSubject` stay.** No signals.
- **`zone.js` stays.** No `provideZonelessChangeDetection`.
- **Karma + Jasmine stay.** No Jest, no Vitest, no Web Test
  Runner.
- **No SSR, no prerendering, no hydration.**
- **No visual changes.** Templates, classes, copy stay.
- **Material 22 prebuilt is acceptable** for v22 even though it
  is deprecated; switching to the M3 `mat.theme` mixin is out of
  scope.
- **Pin with `~`.** Avoid floating to a new minor accidentally.
- **`ng update` must run with Node ≥ 22.22.3 / 24.15.0.** Every
  `ng update` invocation must prepend
  `/opt/homebrew/opt/node@24/bin` to `PATH` (or use the full path
  to `node`) so the schematic's Node check passes.

## Risks

- **`ng update` peer dep failure.** Angular 22 may pull in a
  transitive dependency that conflicts with the existing Tailwind
  / PostCSS versions. Mitigation: pin Tailwind / PostCSS if
  necessary, or use `--legacy-peer-deps` only as a last resort.
- **v22 breaking changes outside our surface.** Detailed v22
  release notes are beyond the model's cutoff. We trust `ng
  update` schematics to apply any breaking-change rewrites in the
  codebase. If a breaking change is not auto-fixed, we apply the
  smallest possible manual fix and document it.
- **TypeScript bump.** Angular 22 may require TS 5.9.x. The
  current code is strict-mode-clean per the existing `tsconfig`;
  the bump should be a no-op for our templates and services.
- **Builder config rename.** We trust `ng update` to perform any
  builder-option renames. If it does not, we do them manually.
- **Material 22 deprecation logs.** Expected and acceptable.
- **Node PATH leak.** Every `ng update`, `tsc`, `ng build`, and
  `ng serve` invocation must use the Node 24 binary. We prepend
  `/opt/homebrew/opt/node@24/bin` to `PATH` for each invocation.
- **Homebrew PATH persistence.** The Homebrew caveat recommends
  adding `node@24` to `~/.zshrc`. We do not modify the user's
  shell config (out of scope; documented as a recommendation).
- **No frontend tests to validate.** We rely on the typecheck +
  build + manual smoke. The contract's "No frontend unit tests"
  invariant is preserved.
- **No CSS regressions expected.** v22 ships zero changes to the
  CSS Custom Properties cascade or to Tailwind's contract.

## Recommendations (do not create in this change)

- **Migrate to Material 22 M3 `mat.theme` mixin.** The prebuilt
  is deprecated; a future change can move to the M3 design token
  system.
- **Adopt standalone components.** A future change can migrate
  `AppModule` → `bootstrapApplication`, `provideRouter`,
  `provideHttpClient`, etc., component by component.
- **Adopt signals.** A future change can rewrite `BehaviorSubject`
  streams as `signal()` / `computed()`.
- **Zoneless change detection.** A future change can flip to
  `provideZonelessChangeDetection` once the streams are signals.
- **Add frontend unit tests.** A future change can add Vitest or
  keep Karma + add specs for `AppService` and `AuthService`.
- **Add a `.nvmrc` / `.node-version` pin.** A future change can
  pin Node 24 in the repo so new contributors pick it up
  automatically.
